# 6.DBマイグレーション

## 6 envファイルを確認

- env

下記の様に環境変数を設定します。

```sh
DATABASE_URL=postgres://{ユーザ名}:{password}@127.0.0.1:{ポート番号}/{DB名}
PAYLOAD_SECRET=YOUR_SECRET_HERE ←プロジェクト作成時に自動的設定される
NEXT_PUBLIC_SERVER_URL={ホスト名}:{ポート番号}
GOOGLE_AI_ST_API=xxxxxxxxxxxxxxxxxx
```

## 6-1.マイグレーション実行

```sh
## 依存パッケージのインストール
npm install

## 新規マイグレーション作成
npm run payload migrate:create

## DB マイグレーションの実行
###まだ実行されていないすべてのマイグレーションを実行
npm run payload migrate
```

## 6-2.マイグレーションのコマンド

- 以下必要に応じて実施。
```sh
## マイグレーション状態
npm run payload migrate:status

## 前回の移行処理をロールバック
npm run payload migrate:down

## 既に実行済みの移行をすべてロールバックし、再度実行
npm run payload migrate:refresh

## データベースからすべてのエンティティを削除し、すべてのマイグレーションを最初から再実行
npm run payload migrate:fresh

```
